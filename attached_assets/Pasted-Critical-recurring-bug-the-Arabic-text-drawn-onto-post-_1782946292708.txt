Technical context (read this before you start)
This project is a bot that publishes tech news to Instagram as image posts. main.py generates a cover image (from DALL-E / gpt-image-1) and then uses Python’s Pillow (PIL) library to draw a large Arabic headline on top of it, using the “Noto Kufi Arabic” font. That font is embedded as a Base64 string inside assets/fonts/font_data.py (not a separate .ttf file) — this was a deliberate, correct fix for an earlier bug where the binary font file kept getting corrupted during Git transfers. Do not revert that part; keep the font embedded as Base64.
The current bug shows up as one of these broken renderings (it has shifted between attempts):
	1.	Empty boxes (“tofu”) instead of letters entirely
	2.	Letters correctly joined/shaped, but the word order is reversed across the whole line
	3.	Most recent: each individual word’s letters appear reversed within the word itself (example: the word “لجهاز” renders as “زاهجل” — literally the letters in reverse order inside that one word), while the word-to-word order looks roughly correct.
Why this keeps happening
I iterated on this from a different development environment (a series of GitHub Pull Requests). Every fix worked in my sandbox but failed differently here. My best guess: this Replit environment is missing libraqm (or has a different/incompatible version of it, or of Pillow, or of FreeType), and correct Arabic complex-text shaping (character joining + BiDi word reordering) depends critically on that library actually being present and functional.
The current code in main.py (see the functions _wrap_arabic_words and _draw_rtl_line near the top of the file) tries to work around this by drawing each Arabic word individually via plain draw.text() (no direction/language kwargs), then manually positioning the words right-to-left by computing coordinates itself. This assumed that rendering a single word with no special parameters would automatically join its letters correctly — that assumption turned out to be wrong specifically in this environment (proof: the letters inside a single word are reversed in the attached screenshot).
